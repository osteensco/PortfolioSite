A web application for creating chatrooms and sending messages.

Server

Live connections are established with the front end clients using websockets and the server itself is buil in Go. Websockets handle sending messages to the server and out to the clients in a given chatroom as well as updating the lobby's room list for clients in the lobby as chatrooms are created in real time. The server has various go routines for receiving and sending messages, checking chatroom population for a clean up protocol, and making API requests. There is also logic for maintaining state across multiple server instances.

Cache and Data Persistance

For data persistence, I utilize a CockroachDB cloud instance for the database and a Redis cloud project as a cache layer. For GET requests, the API will first try to read from the cache and fallback to the database. The REDIS key will then be added if it wasn't already in the cache. For POST, PUT, and DELETE requests, the API will first write to the database. If this is successful, a check is executed on the existance of a corresponding Redis key. If applicaple, the operation will also be performed on the REDIS key. This way, the cache stays up to date with the database.
