Object Oriented Batch Pipeline with CI/CD

I use a python script that is run as a Google Cloud Function, which is continuously deployed by a google cloud build trigged off of push requests to the master branch in the Github repo.

The Cloud Function is run off a daily pub/sub call, the pipeline identifies the DataSources that need to be ingested and executes those while skipping any that don't need to run.

I used an Object Oriented approach for the design of the pipeline. I create a unique child DataSource object for each set of data I am ingesting which are fed through a Pipeline object that handles fetching API keys, scheduling, and general process flow. The DataSource objects are designed to handle the ETL steps required, and land the data into the appropriate table in Google Bigquery.

The pipeline is simple to scale as it only requires a child DataSource object to be created in order to integrate an additional ingestion need.
