Object Oriented Batch Pipeline with CI/CD

For each data source I use a python script in a Google Cloud Function, which is continuously deployed by a google cloud build trigged off of push requests to the Github repo. The code is the same for each, but each data source has it's own object, execution function, and yaml file to define the correct entry point and follow the correct workflow.

The Cloud Function is run off a daily Pub/Sub call from Cloud Scheduler, or is triggered when a Google Cloud Function HTTP Endpoint publishes a message to the Pub/Sub topic that the Cloud Function is subscribed to. The pipeline verifies if it's DataSource needs to be ingested or not to avoid duplicated data and actions appropriately.

I used an Object Oriented approach for the design of the pipeline. I created a unique child DataSource object for each set of data I am ingesting which are fed through a Pipeline object that handles fetching API keys, scheduling, and general process flow. The DataSource objects are designed to handle the ETL steps required, and land the data into the appropriate table in Google Bigquery.

The pipeline is simple to scale as it only requires a child DataSource object and a yaml file to be created in order to integrate an additional ingestion need.
